# kb-system 项目介绍

> 个人知识图谱构建与管理 Agent — 面向非技术背景的汇报

---

## 一、解决什么问题

每天我们和 AI 进行大量对话——分析数据、排查 bug、做技术决策、学新工具。这些对话里藏着大量的**经验和洞察**，但对话结束后就沉没了。下次遇到类似问题，只能重新问一遍。

kb-system 做的事情很简单：**把对话变成可搜索、有关联、能发现的知识库。**

就像给对话装上了搜索引擎和"猜你喜欢"——不只能搜到以前说过什么，还能主动告诉你"这几篇笔记其实在讲同一件事"。

## 二、怎么工作

```
你说的话 → 自动判断值不值得记 → 生成一篇结构化工整的笔记
                                              ↓
                              以后能用大白话搜到（不用记关键词）
                                              ↓
                              系统自动发现这篇笔记和哪些旧笔记有关联
```

### 举个例子

你之前做过一次 GSB 评估分析，讨论了"安全提醒"这个维度的重要性。几周后你问：

> "我之前关于模型回答安全性有什么结论来着？"

系统搜索你的知识库，返回：
- GSB 评估报告（讲安全提醒是猎户座最大优势维度）
- GSB 实战经验（记录了分析方法论）
- 法条 Review 总结（安全相关的判定规则）

**不需要记文件名、不需要打标签、不需要事先整理。自然语言问就行。**

## 三、两种用法

| | 轻量版（Skill） | 完整版（kb-system） |
|---|---|---|
| 适合 | 只想随手记录 | 想建知识大脑 |
| 配置 | 零 | 填一个 API Key |
| 能做什么 | 记录对话 → 笔记 | 记录 + 搜索 + 图谱 + 可视化 |
| 安装 | 复制一个文件 | pip install |

## 四、核心亮点

### 🧠 会关联，不是孤岛

传统的笔记工具（Notion、Obsidian、备忘录）需要你手动加链接、打标签。kb-system 自动完成——它分析每篇笔记的内容，发现：

- "这篇 GSB 报告在讨论安全提醒" ←→ "这篇方法论里也提到了安全评估" → 自动关联
- "今天的对话和上周关于 TailwindCSS 迁移的内容高度相关" → 自动提示

### 🔍 用大白话搜

不需要记"这篇笔记叫 `GSB评估-猎户座vs豆包-Opus与联盟.md`"。问"安全提醒的结论是什么"，系统理解你的意思，找到对的笔记。

### 🤖 不是 Agent 自己记，是第三方记

重要设计：不是让正在和你对话的 AI 去总结对话，而是**交给另一个独立的 AI 来完成**。原因很简单——让作者给自己的书打分会偏高。第三方评判更客观。

### 🔓 数据完全本地

所有笔记、索引、图谱数据都在你自己的电脑上。不联网也能搜。

## 五、适用场景

| 场景 | 说明 |
|------|------|
| 开发者 | 记技术决策、踩坑记录、项目总结，下次遇到直接搜 |
| 研究员 | 文献分析、实验结论、方法论沉淀，发现跨主题关联 |
| 知识工作者 | 会议纪要、方案论证、学习笔记，告别"我好像记过但找不到" |
| 任何人 + AI | 和 ChatGPT/Claude 的对话，自动转化为可复用知识 |

## 六、技术可信度

- ✅ 开源 (MIT)，代码可审计
- ✅ 17 个自动化测试
- ✅ 不依赖特定工具 (Obsidian/AI 平台可替换)
- ✅ 支持 Claude Desktop、Cursor、Hermes 等多种 AI 工具

## 七、总结

kb-system 解决的问题一句话：**你和 AI 的对话里，有多少经验白白流失了？**

它把这些经验变成真正属于自己的知识资产——可搜索、有关联、能发现。
