## 主文案（故事型，一版通用）

我开发了一个**个人知识图谱构建与管理 Agent**。

事情的起因是这样的——

我每天大量使用 AI：分析数据、排查 bug、做技术决策、学新工具。这些对话里藏着大量经验，但聊完就沉了。几周后遇到类似问题，隐约记得"好像讨论过"，翻聊天记录翻到崩溃，最后只能重新问一遍。

于是我开始想两个问题：

**第一，怎么自动记？** 我不想每次聊完还要手动整理笔记——太累了。最好是对话结束就自动沉淀，AI 自己判断值不值得记，归类到哪，生成一篇工整的结构化笔记。

**第二，怎么找得到？** 传统的笔记工具靠文件名和标签检索，但我根本不记得三个月前那篇笔记叫什么。我需要用大白话搜——"之前关于安全提醒的结论是什么"——就能找到对的笔记。

基于这两个需求，我花了几个晚上搭了这个系统。核心思路很简单：

```
对话 → AI 判断价值 → 生成结构化笔记
              ↓
    以后能用大白话搜到（语义搜索）
              ↓
    系统自动发现这篇笔记和哪些旧笔记有关联（知识图谱）
```

几个有意思的设计决策：

**1. 不让 Agent 自己评自己。** 摄入管线用的是独立的外部 LLM，而不是让当前对话的 AI 自己总结自己。同一模型评判自己的输出会有偏见，就像让作者给自己写书评——打分总是偏高。第三方更客观。

**2. 知识图谱自动建。** 新笔记进来，系统会自动发现它和旧笔记的关联——共享的标签、相互引用的 wikilink、语义内容相似度。不需要手动加链接，关联自己长出来。

**3. 不绑定平台。** 读取的是普通 Markdown 文件夹，你可以用 Obsidian 编辑、用 kb-system 搜索。任何 AI 工具（Claude、Cursor、Hermes）通过 MCP 通用协议都能接入。

目前已经开源，17 个测试全过，代码 1800 行。

🔗 github.com/Zola-ops/kb-system

## 精简版（适合即刻/朋友圈）

我开发了一个**个人知识图谱 Agent**。

起因很简单：和 AI 聊完天，经验就沉没了。下次想找，翻聊天记录翻到崩溃。

所以做了这个——对话自动变笔记，大白话就能搜到，系统自动发现笔记间的关联。本地运行，数据完全自己掌控。开源。
