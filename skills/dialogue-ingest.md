# 对话沉淀 Skill

> 将当前对话总结为结构化 Markdown 笔记，写入本地文件夹。零配置，零依赖。

## 适用场景

任何能读写文件的 Agent 工具均可使用。用户说"记录这段对话""沉淀到知识库"即触发。

## 与 kb-system 的关系

| | 本 Skill | kb-system |
|---|---|---|
| 摄入笔记 | ✅ Agent 自己总结 | ✅ 外部 LLM 总结 |
| 配置 | **零** | 需配 API Key + 安装 |
| 语义搜索 | ❌ | ✅ |
| 知识图谱 | ❌ | ✅ |
| 写入位置 | 任意 Markdown 文件夹 | 任意 Markdown 文件夹 |
| 跨平台 | ✅ 纯文本，处处可用 | ✅ MCP |

> **本 Skill = 随手记。kb-system = 知识大脑。**

## 操作步骤

### 1. 判断是否值得记录

回顾对话内容，只有以下场景才记录：
- 完成了复杂任务（3+ 步骤）
- 解决了非预期错误/bug
- 得出了分析结论
- 创建/部署了项目
- 学到了新工具/技术用法
- 做出了重要决策

简单问答、闲聊、查信息 → 不记录。

### 2. 确定分类和标题

| 分类 | 判断标准 |
|------|---------|
| Projects | 创建、部署、开发、构建 |
| Fixes | 修复 bug、排查问题、踩坑 |
| Reports | 分析、报告、评估、总结 |
| Learnings | 学习、发现、新技术 |

标题 15 字以内，简洁概括。

### 3. 生成结构化笔记

按以下模板生成 Markdown，写入笔记文件夹：

```markdown
# {标题}

> {一句话总结}

## 背景

为什么要做这件事？

## 做了什么

### 关键步骤
1. 步骤一
2. 步骤二

### 技术细节
- 使用的工具/技术
- 关键操作

## 踩坑记录

（没有则写"本次无踩坑"）

### 坑：{问题描述}
- **现象**：xxx
- **原因**：xxx
- **解决**：xxx

## 收获与反思

1. **技术收获**：xxx
2. **可复用经验**：xxx

## 相关链接

- [[相关笔记1]]
- [[相关笔记2]]

---
*{日期}*
```

### 4. 写入文件

文件路径：`{笔记根目录}/{分类}/{标题}.md`

笔记根目录优先级：
1. 用户指定的路径
2. `~/Documents/Obsidian Vault`（Obsidian 默认）
3. `~/notes`（通用 fallback）

### 5. 更新索引（可选）

如果该文件夹有 `Daily/` 目录结构，在当天日记中追加 wikilink：
```
- [[笔记标题]] — 一句话摘要
```

### 6. 告知用户

输出：分类、标题、文件路径。

## 注意事项

- 只记录对话中真实发生的事情，不编造
- 踩坑记录要有现象→原因→解决的完整链路
- 标题用中文，简洁有意义
- 如果对话属于"简单问答"，诚实说"这不值得记录"
